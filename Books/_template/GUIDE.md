# Book 作成ガイド

このテンプレートを使って新しい学習ガイド（Book）を作成する方法を説明します。

## 📝 新しい Book の作成手順

### 1. ディレクトリの作成

```bash
# Books ディレクトリに移動
cd Books

# 新しい Book のディレクトリを作成
mkdir Your-Topic-Name
cd Your-Topic-Name
```

### 2. テンプレートのコピー

```bash
# テンプレートから必要なファイルをコピー
cp ../_template/00-Introduction.md .
cp ../_template/README.md .
```

### 3. ファイル名の規則

章ごとのファイルは以下の命名規則に従ってください:

```
00-Introduction.md          # 必須: イントロダクション
01-Basics.md               # 基本概念
02-Core-Concepts.md        # コア概念
03-Advanced-Topics.md      # 応用トピック
99-Appendix.md             # 付録（オプション）
```

**命名のポイント**:

- 2 桁の連番 + ハイフン + タイトル
- タイトルはケバブケース（単語を `-` で連結）
- 00 は必ず Introduction にする

### 4. README.md の編集

各 Book の README には以下を含めてください:

```markdown
# [Book のタイトル]

## 📚 概要

[1-2 文で何を学べるか]

## 🎯 学習目標

- 目標 1
- 目標 2

## 📖 目次

- [章のリスト]

## 📋 前提知識

[必要な前提知識]

## ⏱️ 推定学習時間

約 X 時間

## 🛠️ 必要な環境

[セットアップ手順]
```

### 5. 各章の構成

各章は以下の構造を推奨します:

```markdown
# [章番号] - [タイトル]

## 概要

[この章で学ぶこと]

## セクション 1

### 概念の説明

### コード例

### 実践課題

## セクション 2

...

## まとめ

[重要なポイントの復習]

## 次のステップ

➡️ 次へ: [次の章へのリンク]
```

### 6. コード例のフォーマット

````markdown
## コード例

```typescript
// ✅ 良い例
const goodExample = () => {
  // コード
};

// ❌ 悪い例
const badExample = () => {
  // コード
};
```
````

### 7. チェックリスト

新しい Book を作成する際のチェックリスト:

- [ ] ディレクトリ名はケバブケースで明確
- [ ] README.md を作成し、目次を整理
- [ ] 00-Introduction.md を作成
- [ ] 各章のファイル名が規則に従っている
- [ ] 各章間のナビゲーションリンクが設定されている
- [ ] コード例は動作確認済み
- [ ] 推定学習時間を記載
- [ ] 前提知識を明記
- [ ] 関連リンクを追加

## 📐 Book の構成パターン

## 🎯 ベストプラクティス

### 内容について

1. **段階的に学習できる構成**: 簡単な概念から徐々に難易度を上げる
2. **実践的な例**: 理論だけでなく、実際に動くコードを含める
3. **視覚的な説明**: 必要に応じて図やフローチャートを追加
4. **チェックポイント**: 各章の終わりに理解度を確認できる課題を設ける

### フォーマットについて

1. **一貫性**: すべての章で同じフォーマットを使用
2. **ナビゲーション**: 各章の最後に前後の章へのリンクを配置
3. **コードブロック**: 言語を指定してシンタックスハイライトを有効化
4. **見出しレベル**: H1 は章タイトルのみ、H2 以降で内容を構造化

### メンテナンスについて

1. **更新履歴**: README に更新履歴を記録
2. **ステータス**: Book の完成度をステータスで明示
3. **レビュー**: 定期的に内容を見直し、最新の情報に更新
4. **フィードバック**: 学習者のフィードバックを反映

## 📊 Book のステータス

README の最後に現在のステータスを明記:

- 🚧 **作業中**: まだ執筆中
- 📝 **レビュー中**: 内容のレビュー待ち
- ✅ **完了**: 学習可能な状態
- 🔄 **更新中**: 既存の内容を更新中
